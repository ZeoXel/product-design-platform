# AI 产品设计台 - 使用场景示例

本文档展示产品设计台的典型使用场景和操作流程。

---

## 场景一：从图库选择参考图生成新设计

### 用户角色
设计师想要基于已有的热销产品进行改款设计

### 操作流程

```
1. 点击「从图库选择」打开图库抽屉
2. 浏览图库中的参考图，点击选择一个海洋风钥匙扣
3. 系统自动分析图片，显示：
   - 风格识别卡片：🐚 海洋风少女系
   - 元素识别：薄荷绿海豚吊坠、奶白色海星吊坠、玻璃珠×12
   - 五金配件：合金龙虾扣、金属旋转扣环
4. 在对话框输入修改需求：「把海豚换成贝壳，颜色改成粉色系」
5. 系统生成新设计，显示在预览区
6. 查看成本估算和生成的设计稿
```

### 系统响应示例

**分析消息**：
> 已选择「海洋风少女系」风格的参考图。检测到元素：薄荷绿海豚吊坠、奶白色海星吊坠。请告诉我你想要的修改。

**生成后消息**：
> 已完成「把海豚换成贝壳，颜色改成粉色系」的修改。你可以继续调整，或点击「导出」保存设计。

## 场景二：上传外部图片进行分析和改款

### 用户角色
买手发现竞品图片，想要分析并生成类似但有差异化的设计

### 操作流程

```
1. 点击上传区域或拖拽图片到上传区
2. 系统自动分析上传的图片：
   - 显示分析进度动画
   - 识别主体元素、辅助元素、五金配件
   - 检测风格类型
   - 查找相似产品（显示相似度百分比）
3. 查看分析结果和相似产品
4. 输入设计指令：「保持整体结构，把珠子颜色改成渐变紫色」
5. 等待生成完成
6. 对比原图和生成图，进行进一步调整
```

### 风格检测示例

上传一个波西米亚风格的包挂后，系统可能识别为：

**风格卡片**：
```
🌿 波西米亚民族风
流苏、编织绳、天然石，大地色系
推荐配色：earth tones, terracotta, cream
```

## 场景三：纯文字描述生成设计

### 用户角色
设计师脑海中有创意想法，想快速可视化

### 操作流程

```
1. 不上传任何图片
2. 直接在对话框输入描述：
   「设计一个星空主题的钥匙扣，有金色星星和月亮吊坠，
    搭配天蓝色小珠子，整体风格梦幻清新」
3. 系统识别风格为「梦幻星空系」
4. 生成设计稿
5. 根据生成结果进行调整：
   「星星再大一点，增加几颗粉色珠子」
```

### 生成的 Prompt 示例

系统内部生成的英文 Prompt：
```
A dreamy starry keychain charm with golden star and moon pendants as focal
points, decorated with sky blue small round beads, silver keyring and lobster
clasp attachment, celestial color palette with gold sky blue and soft pink,
minimalist kawaii style, product photography, pure white background, soft
ethereal lighting, 15cm total length
```

## 场景四：切换风格进行跨风格改款

### 用户角色
设计师想将一个海洋风产品改成波西米亚风格

### 操作流程

```
1. 上传或选择一个海洋风产品
2. 系统识别为「海洋风少女系」
3. 点击风格卡片，系统提示：
   「已切换到「波西米亚民族风」风格。
    流苏、编织绳、天然石，大地色系。
    推荐配色：earth tones, terracotta, cream。」
4. 输入指令：「按照波西米亚风格重新设计，保留基本结构」
5. 系统结合新风格生成设计
6. 对比两种风格的设计效果
```

### 风格对比

| 原风格（海洋风） | 新风格（波西米亚） |
|-----------------|-------------------|
| 贝壳、海星吊坠 | 天然石、流苏吊坠 |
| 马卡龙色系 | 大地色系 |
| 玻璃珠装饰 | 编织绳装饰 |
| 清新甜美 | 自然复古 |

## 场景五：迭代优化设计

### 用户角色
设计师需要多次调整直到满意

### 操作流程

```
第一轮：
1. 上传参考图
2. 输入：「增加一个心形吊坠」
3. 生成 V1 版本

第二轮：
4. 查看 V1，输入：「心形吊坠太大了，缩小一半」
5. 生成 V2 版本

第三轮：
6. 查看 V2，输入：「把金属件改成玫瑰金色」
7. 生成 V3 版本

最终：
8. 通过版本栏切换查看 V0→V1→V2→V3
9. 选择最满意的版本导出
```

### 版本管理

底部版本栏显示所有历史版本：
```
[原图 V0] → [V1 +心形] → [V2 缩小] → [V3 玫瑰金] ← 当前
```

## 场景六：相似产品参考

### 用户角色
设计师想了解市场上类似的产品

### 操作流程

```
1. 上传或选择参考图
2. 系统分析后，在「相似」区域显示：
   - 产品A (相似度 85%)
   - 产品B (相似度 72%)
   - 产品C (相似度 68%)
3. 点击相似产品查看详情
4. 参考相似产品的设计元素进行创新
5. 输入：「结合产品A的配色方案」
```

### 相似度算法

系统基于以下维度计算相似度：
- 元素类型匹配
- 风格标签重合
- 结构类型相似
- 配色方案接近

## 最佳实践

### 输入指令技巧

| 好的指令 | 不好的指令 |
|---------|-----------|
| 「把主元素换成粉色贝壳」 | 「换一下」 |
| 「增加3颗透明水晶珠」 | 「加点珠子」 |
| 「整体配色改成马卡龙色系」 | 「换个颜色」 |
| 「保持海洋风格，把海星改成海马」 | 「改改」 |

### 风格一致性

- 修改时尽量保持在同一风格体系内
- 跨风格改款时明确说明目标风格
- 使用风格选择器切换风格会获得更好效果

### 元素数量控制

- 主元素建议 1-3 个
- 辅助元素建议 5-15 个
- 五金配件建议 2-4 个
- 元素过多会增加成本和制作难度
