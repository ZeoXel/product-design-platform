## 第一页：行业痛点分析

**标题**: 现有 AI 设计工具的根本缺陷

### 通用 AI 绘图的困境

```
传统模式:
用户 ──→ 提示词 ──→ 黑盒模型 ──→ 结果
              ↑ 断裂点
用户不满意，只能重新写提示词
知识无法沉淀，每次从零开始
```

### 挂饰设计的特殊挑战

| 挑战类型 | 具体表现 |
|---------|---------|
| **物理约束** | 材料属性、工艺可行性、承重限制 |
| **风格一致性** | 元素搭配协调、整体美感把控 |
| **商业适配** | 目标人群匹配、季节趋势把握 |
| **迭代效率** | 反复修改耗时、沟通成本高 |

**讲解词**:
> "通用 AI 绘图工具有一个根本问题：它是'一次性'的。每次生成都是独立的，系统不会记住您的偏好，不会学习什么样的设计更受欢迎。我们要解决的正是这个问题——构建一个会学习、会进化的设计系统。"

## 第二页：解决方案概览

**标题**: 自进化设计系统 —— 范式转变

### 核心理念：从"单次生成"到"持续学习闭环"

```
┌─────────────────────────────────────────────────────────┐
│                     自进化系统架构                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│   用户意图 ──→ 知识检索 ──→ Prompt编排 ──→ 图像生成    │
│      ↓                                         ↓        │
│      └──── 选择行为 ←───── 质量评估 ←──────────┘        │

| 层级 | 功能 | 核心能力 |

## 第三页：双模式设计能力

**标题**: 精确修改 + 探索创新，双轮驱动

### 模式一：图生图 —— 基于参考的精确编辑

```
适用场景: 已有设计基础，需要局部调整

工作流程:
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  上传参考图  │ ──→ │ 对话式修改   │ ──→ │  精确输出   │
│  (您的产品)  │     │ "换成蓝色"  │     │ (保留结构)  │
└─────────────┘     └─────────────┘     └─────────────┘

核心优势:
  ✓ 继承参考图的物理结构和工艺可行性
  ✓ 只修改指定部分，其他保持不变
  ✓ 渐进式迭代，2-3次达到满意
```

### 模式二：文生图 —— 智能参考匹配探索

```
适用场景: 创意探索阶段，不确定具体方向

工作流程:
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 描述需求    │ ──→ │ 智能匹配参考 │ ──→ │ 多方案输出  │
│"海洋风少女感"│     │ 从图库检索   │     │ (供选择)    │
└─────────────┘     └─────────────┘     └─────────────┘

核心优势:
  ✓ 自动从图库检索最匹配的参考图
  ✓ 结合参考特征生成新设计
  ✓ 输出多个差异化方案供探索
  ✓ 兼顾创意发散和可行性约束
```

### 双模式协同

```
典型工作流:

阶段1: 探索 (文生图模式)
  "我想要一款海洋风的少女系挂饰"
  → 系统自动匹配参考，生成 4 个差异化方案
  → 用户选择方案 B 作为基础

阶段2: 精修 (图生图模式)
  基于方案 B 进行对话式修改:
  → "主体元素再大一点"
  → "加一些珍珠点缀"
  → "配色再温柔一些"
  → 最终确认输出
```

**讲解词**:
> "我们的系统支持两种工作模式。当您已有设计基础想要调整时，使用图生图模式进行精确编辑；当您在探索阶段不确定方向时，使用文生图模式——系统会自动从图库中检索匹配的参考，结合参考特征生成多个方案供您选择。两种模式可以无缝衔接：先探索找到大方向，再精修细节。"

## 第四页：对话式精确编辑

**标题**: 自然语言控制，精确到每个元素

### 对话式迭代流程

```
👤 用户: "把粉色贝壳换成水晶"
    ↓
🔍 Agent 解析:
    {
      action: "replace",
      target: "seashell",
      replacement: "crystal",
      constraints: {keep_position: true, match_style: true}
    }
    ↓
📚 知识库查询:
    • crystal 有哪些变体？ → [水晶柱, 水晶球, 碎水晶...]
    • crystal + starfish 兼容性？ → 0.85 (良好)
    • 物理约束检查 → 通过
    ↓
✏️ Prompt 编排:
    根据查询结果，动态生成最优 Prompt
    ↓
🖼️ 生成图片 v1

👤 用户: "水晶大一点，更闪亮些"
    ↓
🔍 Agent 解析:
    {
      action: "modify",
      target: "crystal",
      changes: {scale: 1.3, brightness: +20%}
    }
    ↓
🖼️ 基于 v1 精确调整，生成 v2
```

### 语义理解能力

| 用户表达 | Agent 解析 | 执行操作 |
|---------|-----------|---------|
| "贝壳大一点" | scale(seashell, 1.2~1.4) | 只放大贝壳，保持其他元素 |
| "整体更温暖" | adjust_color_temperature(+15%) | 调整色调，不改变结构 |
| "换成蓝色系" | recolor(target: all, palette: blue_family) | 智能替换配色方案 |
| "简化一下" | reduce_complexity(level: -1) | 移除次要装饰元素 |
| "更精致一些" | enhance_detail(quality: +1) | 提升细节精细度 |
| "风格再甜一点" | adjust_style(kawaii: +0.2) | 增加可爱元素特征 |

**讲解词**:
> "您不需要学习任何专业术语。直接用日常语言描述修改需求，Agent 会自动理解并解析成结构化指令，查询知识库确认可行性，然后编排最优的 Prompt 执行修改。每次修改都在上一版基础上进行，不会丢失之前满意的部分。"

### 从"图片集合"到"知识资产"

```
传统图库:                       我们的结构化图库:
┌─────────────┐                ┌─────────────────────────────┐

│ image_002.jpg │               │ ├─ image.jpg    ← 视觉资产  │
│ image_003.jpg │               │ ├─ elements: [贝壳,海星,珍珠]│

                               │ ├─ tags: {季节,人群,场景}  │
                               │ └─ prompt_template: "..."   │
                               └─────────────────────────────┘

### 智能检索机制

```
用户输入: "海洋风格，粉色系，少女感"

  3. 色彩分析匹配 → 筛选粉色系为主的
  4. 业务标签匹配 → 目标人群=少女

**讲解词**:

**标题**: 不是固定模板，而是智能组装

```
┌─────────────────────────────────────────────────────────┐
│                    Prompt 基因库                         │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  主题描述块 (Theme Genes)                                │
│  ├─ ocean_theme: "ocean inspired, seashells, starfish"  │
│  ├─ crystal_theme: "crystal elements, gemstone..."      │
│  ├─ vintage_theme: "retro style, antique finish..."     │
│  └─ ...                                                  │
│                                                          │
│  风格描述块 (Style Genes)                                │
│  ├─ kawaii: "cute, adorable, soft rounded shapes"       │
│  ├─ minimalist: "clean lines, simple, modern"           │
│  ├─ bohemian: "free-spirited, natural materials"        │
│  └─ ...                                                  │
│                                                          │
│  质量控制块 (Quality Genes)                              │
│  ├─ lighting_A: "studio lighting, soft shadows"         │
│  ├─ lighting_B: "natural daylight, outdoor feel"        │
│  ├─ detail_high: "intricate details, 8K resolution"     │
│  └─ ...                                                  │
│                                                          │
│  元素描述块 (Element Genes)                              │
│  ├─ seashell: "natural seashell, pink tones..."         │
│  ├─ crystal: "raw crystal point, translucent..."        │
│  └─ ...                                                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 动态编排流程

```
用户需求: "海洋风 + 可爱风格 + 水晶元素"
    ↓
Agent 分析:
  • 主题 = ocean_theme
  • 风格 = kawaii
  • 元素 = crystal (替换原有元素)
    ↓
知识库查询:
  • ocean_theme + kawaii 最佳组合模板？ → template_023
  • crystal 的最优描述词？ → "raw rose quartz crystal point"
  • 历史成功率最高的质量控制块？ → lighting_A + detail_high
    ↓
Prompt 组装:
  ┌────────────────────────────────────────────────────┐
  │ ocean_theme.base                                    │
  │ + kawaii.style_modifier                             │
  │ + crystal.element_description                       │
  │ + lighting_A + detail_high                          │
  │ + 参考图特有的成功因子                              │
  └────────────────────────────────────────────────────┘
    ↓
最终 Prompt: "A kawaii ocean-inspired charm keychain
             with raw rose quartz crystal point replacing
             the seashell, maintaining soft rounded shapes,
             studio lighting with soft shadows,
             intricate details, 8K, commercial photography"
```

**讲解词**:
> "我们的 Prompt 不是固定模板，而是由多个'基因块'动态组装的。系统会根据您的需求，从主题库、风格库、元素库、质量控制库中选择最合适的模块，结合参考图已验证的成功因子，智能组装成最优 Prompt。这种机制确保了生成质量的稳定性和可控性。"

## 第八页：知识图谱与兼容性检查

**标题**: 设计决策的智能约束

### 元素知识图谱

```
┌─────────────────────────────────────────────────────────┐
│                     元素知识图谱                         │
├─────────────────────────────────────────────────────────┤
│                                                          │
│            ┌─────────┐                                  │
│            │ seashell │                                  │
│            └────┬────┘                                  │
│         ┌──────┼──────┐                                 │
│         ↓      ↓      ↓                                 │
│    ┌────────┐ ┌────┐ ┌────────┐                        │
│    │starfish│ │pearl│ │crystal │                        │
│    │兼容:0.95│ │兼容:0.92│ │兼容:0.78│                   │
│    └────────┘ └────┘ └────────┘                        │
│                                                          │
│  兼容性规则:                                             │
│  • seashell + starfish → 经典海洋组合 (0.95)            │
│  • seashell + pearl → 优雅搭配 (0.92)                   │
│  • seashell + crystal → 风格冲突风险 (0.78)             │
│                                                          │
│  物理约束:                                               │
│  • 总重量 < 挂钩承重 × 0.8                              │
│  • 元素尺寸比例 1:0.3~0.7                               │
│  • 材质硬度差异 < 3 级                                   │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 兼容性检查流程

```
用户请求: "把贝壳换成大水晶"
    ↓
Agent 查询知识图谱:
  1. 元素兼容性: crystal + starfish = 0.78 (中等)
  2. 风格一致性: crystal 偏现代，原图偏自然 → 风险提示
  3. 物理约束: 大水晶重量 25g > 挂钩承重 20g × 0.8 → 超标
    ↓
Agent 响应:
  ┌────────────────────────────────────────────────────┐
  │ ⚠️ 兼容性提示                                       │
  │                                                     │
  │ 检测到潜在问题:                                     │
  │ • 水晶与现有海星风格差异较大 (兼容度 78%)           │
  │ • 大水晶可能超过挂钩承重限制                        │
  │                                                     │
  │ 建议方案:                                           │
  │ A. 使用小号水晶 (解决承重问题)                      │
  │ B. 使用海洋风水晶(蓝色/透明) (提升风格兼容)        │
  │ C. 同时更换海星为更现代的元素                       │
  │                                                     │
  │ [继续原方案] [采用建议A] [采用建议B]                │
  └────────────────────────────────────────────────────┘
```

### 知识图谱自动更新

```
数据来源:
  • 用户选择行为 → 元素组合偏好
  • 生成质量评分 → 组合效果验证

## 附录：技术问答

### Q1: 文生图模式下，系统如何保证设计的可行性？

> A: 文生图不是凭空生成，而是基于图库检索最匹配的参考图，结合参考的成功经验和 Prompt 模板生成新设计。参考图都是经过验证的真实产品，系统在此基础上创新，既保证创意发散，又有可行性约束。

### Q2: Prompt 模板是固定的吗？

> A: 不是。我们采用"基因库"机制，Prompt 由多个模块动态组装。系统会根据需求选择最优的主题块、风格块、质量块等，结合参考图的成功因子，智能编排最终 Prompt。而且这些基因块的权重会根据生成效果持续优化。

### Q3: Agent 自进化会不会导致系统不稳定？

> A: 我们有三层防护：核心规则需人工审核、组合规则需统计检验、只有动态参数全自动更新。同时有金丝雀发布、异常回滚等安全机制。确保系统稳定进化，不会因错误学习退化。

### Q4: 知识图谱的兼容性规则是怎么来的？

> A: 初始规则由行业专家标注，后续通过用户行为数据自动优化。比如某个元素组合被大量用户选择且评分高，系统会提升其兼容性分数；反之则降低。这样兼容性规则会越来越符合实际市场偏好。

### Q5: 系统能学习特定客户的审美偏好吗？

> A: 可以。系统会记录每个客户的选择行为、修改模式、评分偏好，构建客户专属的偏好模型。随着使用增加，系统对该客户的理解越深，推荐和生成越精准。
